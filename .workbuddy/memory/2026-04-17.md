# 2026-04-17

## Modem Cat 桌面端调试修复

### 问题
COM口连接后页面只有SIM状态显示（READY），其余所有字段（IMEI、ICCID、运营商、RSRP等）全部为 `--`。

### 根因分析
1. **串口超时过短**：`SerialTransport::new()` 设置 `timeout(500ms)`，导致 AT 指令响应读取时频繁超时
2. **drain 逻辑阻塞**：`send_at()` 中 drain 缓冲区时没有超时限制，可能卡在无限阻塞的 read
3. **前端字段名不匹配**：Rust `serde(rename_all = "camelCase")` 序列化后 `cell_id` → `cellId`，但前端用的是 `s.cell_id`（snake_case），导致数据虽然拿到了但无法映射到 DOM 元素
4. **AT 指令间无延迟**：连续快速发送 10+ 条 AT 指令，模组可能来不及处理
5. **错误静默吞掉**：`query_modem_status` 中用 `if let Ok(...)` 吞掉错误，没有日志记录

### 修复内容
- **transport.rs**: 串口超时 500ms → 2s；drain 增加超时限制；read_response 总超时 5s → 8s；drain/response 切换超时时临时修改端口 timeout
- **at_adapter.rs**: AT+CCID 失败时 fallback 到 AT+QCCID；每条指令间增加 100ms 延迟；所有 if let Ok 改为 match 并添加 warn 日志
- **index.html**: 前端字段名全部改为 camelCase 匹配 Rust 序列化（cellId, ipv4Addr, antValues 等）；refreshAll/refreshModemStatus 增加终端日志输出便于调试
